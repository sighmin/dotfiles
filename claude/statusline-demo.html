⚡

Instant prompt

Zero network on the render path. A detached refresh keeps service status fresh in a 60-second cache — your prompt never waits on curl.

🚦

Status-aware model

The model name recolours with status.claude.com — but only for incidents that actually name your model. No false alarms from an unrelated outage.

📉

Budget meters

cx: tracks context remaining; 5h: tracks your 5-hour window. Both ramp dim → bold → yellow → red as you near the edge.

✂️

Smart shortening

Long branches truncate cleanly. Linear / Jira-style refs keep their ticket number and drop the prose. Worktrees replace the branch when you're in one.

    Get it

    Hand it to your agent

    The whole setup lives in one Markdown file — two scripts and a settings merge. Drop it next to your agent and ask. It does the rest.

    1. Fetch the guide into your home directory:

    # grabs the self-contained how-to (scripts embedded inline)
curl -fsSL https://raw.githubusercontent.com/sighmin/dotfiles/main/claude/statusline-setup.md \
  -o ~/statusline-setup.md

    2. Open Claude Code in your home directory and say:

    "Read statusline-setup.md and set up this statusline."

    Your agent writes ~/.claude/statusline-command.sh and ~/.claude/status-refresh.sh, merges the statusLine block into your settings (non-destructively), and verifies the render. It shows up on your next prompt.

    Prefer to read first? See the setup guide and the scripts on GitHub.

    Built by svd · runs entirely on status.claude.com's public summary · no telemetry, no build step, one file.
